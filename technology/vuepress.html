                # VuePress(1) - 初期設定

                # vuepressとは

                vuepressは、CMS(コンテンツ管理システム)の1種である。 markdown形式で書かれた文書をもとに、非常に軽量なウェブサイトを自動で作ってくれる仕組みと考えれば良い。まさにこのウェブサイトがvuepressで作ったものである。
                検索機能、メニューバー等は、初期設定さえ済ませれば投稿記事に連動して自動で更新されるので、更新がとても容易になる。

                なお、以降の記事はMac環境を前提として記載している。Windowsの方はごめんなさい。

                # 導入手順

                1. 環境の整備
                  • nodebrewのインストール
                  • vuepressのインストール
                  • yarnのインストール
                2. ローカルで起動
                  • README.mdを作成する
                  • buildして開発環境で起動する
                3. git pagesで公開
                  • githubの登録を行う
                  • repositoryを作成する
                  • vuepressで作成したページwpuploadする
                4. githubとローカルデータの自動連携
                  • コマンドラインでアップロード(push)する
                  • スクリプト化する



                # 1. 環境の整備

                homebrewのインストール
                homebrewをまだインストールしていない場合、こちらから実施する。
                homebrew (opens new window)

                nodebrewのインストール
                nodebrewをインストールする

                brew install nodebrew

                PATHを通す

                echo 'export PATH=$HOME/.nodebrew/current/bin:$PATH' >> ~/.bash_profile
source ~/.bash_profile
if [ -f ~/.bashrc ] ; then
. ~/.bashrc
fi

                nodebrew -v でバージョンが表示されたらOK

                vuepressをインストールする
                vuepressをインストールする

                npm install --save-dev vuepress

                package.jsonを生成する

                npm init -y

                yarnをインストールする

                npm install -g yarn



                # 2. ローカルで起動

                README.mdを作成する。このファイルに書いた内容がトップページ(index.html)になる。

                mkdir src && echo '# Hello VuePress' > doc/README.md
mkdir src/.vuepress

                vi package.json でファイルを編集し、scriptsのセクションを以下に変更する。
                開発環境をどこで起動し、デプロイしたファイルをどこに書き出すかが設定される。

                "scripts": {
    "dev": "vuepress dev doc",
    "build": "vuepress build dist"
  },



                buildして開発環境で起動する

                markdown形式のファイルをHTMLに変換する操作をbuildという。buildを実行する。

                yarn build

                buildを実行すると、/dist/フォルダが生成され、その中に HTML/CSS/JavaScript のファイルが生成される。
                書き出し先を変更したい場合は、package.jsonに書いた「dist」の部分を変更すれば良い。


                開発環境を起動する
                自分のパソコンの中に、サーバ機能を立ち上げるイメージ。

                npm run dev

                開発環境の起動に成功すると、プロンプトに下記のような結果が表示される。

                success [23:10:28] Build 791d68 finished in 3282 ms! 
> VuePress dev server listening at http://localhost:8080/

                書かれている通り、localhost:8080 にアクセスすると、「Hello VuePress」のサイトが開く。

                # 3. git pagesで公開

                githubの登録を行う
                webサイトの指示に従って登録を行う。
                https://github.co.jp/ (opens new window)

                repositoryを作成する
                新しいリポジトリを作成する。 好きな名前で良いが、公開するWEBサイトのアドレスに入るので、よく考えるべき。

                vuepressで作成したページをuploadする
                distフォルダの中身を全て選択し、作成したリポジトリの中にアップロードすれば、git pagesとして公開される。
                なお、githubには容量500MBまでという上限もあるようなので、むやみに画像や動画を使わず、なるべく軽量なウェブサイトを作ることが望ましい。

                # 4. githubとローカルデータの自動連携

                なぜ自動連携？
                ローカル(PC内)で記事を更新した後、GUI操作だと以下のような操作が発生する。

                • ブラウザを開いてお気に入りからGithubを開く
                • Githubにログインする
                • アップロードの画面を開く
                • Finderを開いてdistフォルダを全部選択する
                • アップロードする
                • commitする

                操作に慣れていても1分近くかかるのではないだろうか。これを数秒で実現できたら、記事を書くモチベーションも高まると考えられる。

                ssh接続の準備

                cd ~/.ssh
ssh-keygen -t rsa

                質問は空欄でEnter3回押せば良い

                id_rsa と、 id_rsa.pub のファイルが生成される。
                .pubファイルの中身をクリップボードにコピーする。

                pbcopy < ./id_rsa.pub

                githubのssh設定画面 (opens new window) を開いて、公開鍵を設定する。 titleは好きな言葉でOK。
                Keyにクリップボードの情報を貼り付けする。

                sshで接続
                以下のコマンドで接続確認を行う。

                ssh -T git@github.com

                接続に成功すれば、以下のように表示される

                Hi koz-sec! You've successfully authenticated, but GitHub does not provide shell access.



                コマンドでアップロード
                distフォルダに移動して、gitを起動する

                cd /Users/koz/vuepress/dist
git init

                githubのデフォルトブランチ名はmainなので、それに合わせてブランチ名を変更する

                git branch -m main

                作ったファイルをローカル(PC側)でコミットする

                git add.
git commit -m "ここにコメント"

                リモート(github側)にプッシュ

                git remote add origin main
git remote set-url origin git@github.com:koz-sec/dxsec.git
git push -f origin main

                Warning

                なお、pushのオプションである -f は強制上書きのオプションなので、複数人でgithubを共用する環境では使用に注意する必要がある。



                スクリプト化する

                というわけで、ここまでのコマンド内容を、vi blog_update.shのような感じで、シェルスクリプトを作って記述すれば良い。

                #!/bin/sh

cd /Users/koz/vue/dist
yarn build

git init
git branch -m main
git add .
git commit -m batch-update

git remote add origin https://github.com/koz-sec/dxsec.git
git push -f origin main

                ここまでできれば、シェルをたまに実行するだけでgit pageが最新化される。
