            # AWS(2) Lambda

            # Lambdaとは

            Lambda(ラムダ)は、サーバレスでコードを実行できるAWSの機能である。 使用した実行時間 に応じて請求が発生する。実行していない時間はお金が掛からないので、コストを必要最小限に抑えることができる。


            # Lambdaの料金

            AWSの無料枠には 契約から1年間のみ無料 のものと、永久無料 のものがある。 Lambdaは、一定水準までなら 永久無料 で使うことができる。

            具体的な価格についてAWSのサイト内に記載がある。そのうち無料の部分について確認する。

            AWS Lambda の無料利用枠には、1 か月あたり 100 万件の無料リクエストと、1 か月あたり 40 万 GB-s のコンピューティングタイムが含まれており、x86 および Graviton2 プロセッサの両方を搭載した機能を利用できます。
            (引用元) AWS Lambda 料金 (opens new window)


            つまり、①リクエスト数 と ②実行時間 の2つで計測される。

            • ①リクエスト数
              永久無料枠は、100万件/月である。 コード実行を行った回数のことである。

            • ②処理時間
              永久無料枠は、40万GB秒/月分のリソース量である。 Lambdaでは、コードを実行するマシンの性能を選択することができる。 1GBメモリの設定であれば、40万秒(約111時間/月)使える。 512MBメモリ(1GBの半分)の設定であれば、80万秒(約222時間/月)使える。


            # Lambda関数の作成

            • 画面左上の サービス を開いて、コンピューティング の中に Lambda がある。

            • Lambdaのページが開いたら、関数の作成 を開く。
            • 一から作成 を選択肢、関数名 に好きな名前を入力、ランタイムでは Python を選択する。

            • 画面下にスクロールして 関数の作成 を押すと、関数が作成される。

            # Lambdaの基本設定

            料金に直結する処理性能と処理時間の設定を行う。

            • Lambda関数の画面で 設定 を開き、一般設定 を開く。
            • メモリ に適切な量を設定する。
              メモリは処理性能に相当する。多いほど処理スピードが早い分、時間あたりコストが高い。
            • タイムアウト時間(起動してから強制終了するまでの時間)を設定する。
              初期値はわずか3秒なので、コードの処理時間が長いものは値を変更して対応する。無駄な時間が少ないようにする。

            # コードの入力

            コード画面を開くと、プログラムコードを書くことができる。

            • 画面を一番下にスクロールして、ランタイム設定 を確認する。
            • Lambda自動時に実行される関数は ハンドラ に書いてある。起動時のコードを変更する場合は、この値も併せて変更する。

            • あとは画面に直打ちでも良いし、作った.pyファイルをアップロードでも良い。

            # 自動実行設定

            トリガーを設定する。

            • 設定、トリガー の順に開き、トリガーを追加を選択する。

            • EventBridgeを検索して選ぶ。

            • 定期実行のルール設定を行う。
            • ルール名 と ルールの説明 には、任意の内容を入れる。
            • スケジュール式 を選び、cron式を書く。
              （図の設定は、毎日 朝7時〜夜22時まで1時間おきにコードを実行する という意味である。）

            • 設定した結果、関数ページのトップにEventBridgeが表示されたら設定完了である。
