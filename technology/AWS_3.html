          # AWS(3) DynamoDB

          # DynamoDBとは

          サーバーレスのデータベースである。 SQLを使わない NoSQL という形態を取る。 管理をAmazonが実施してくれる点がメリットである。

          (参考) DynamoDB (opens new window) / AWS


          # DynamoDBの料金

          AWSの無料枠には 契約から1年間のみ無料 のものと、永久無料 のものがある。 DynamoDBは、一定水準までなら 永久無料 で使うことができる。

          永久無料かどうかは、①ストレージの容量、②データ転送頻度、③データ転送量 の3つで計測される。

          • ①ストレージの容量
            永久無料枠は、25GB/月までである。

          • ②データ転送頻度
            永久無料枠は、読み書きそれぞれ 25ユニット/1時間 である。 1ヶ月ごとにユニット合計使用量が計測され、超過分は有償である。
            この ユニット という単位がやや複雑で、丸めた表現では下記の通り。
            • 書き込みが1秒間に1回発生することを 1WCU(Write Capacity Unit)という。
            • 読み取りが1秒間に1回発生することを 1RCU(Read Capacity Unit)という。
            • 転送データ量が多い時は 1ユニットではなく、複数ユニットを消費する。

          • ③データ転送量
            永久無料枠は、データの取り出しと書き込みで料金が異なる。
            • DBに対するデータの書き込みは、無料である。
            • DBからデータの読み取りは、1GB/月が無料だが、超過分は有償である。

          # DynamoDBの作成

          • 画面左上の サービス を開き、データベース から DynamoDB を選択する。

          • DynamoDBのページが開いたら、テーブルの作成 を選択する。
          • 好きな テーブル名 と、キーになる項目の名前を パーティションキー に入力する。
          • ソートキー はデータの並び順なので、日付等を設定する。

          • 下にスクロールするとテーブル設定がある。デフォルト設定 を選ぶ。
          • さらに下へスクロールして、テーブルの作成 を押す。

          • DBの作成までしばらく時間がかかる。1分ほど待っていると、テーブルが正常に作成された旨が表示される。

          # DynamoDBを操作するための基本設定

          データベースにアクセスする権限を設定する。

          • 一度DynamoDBの画面を離れ、IAM を開く。

          • 左メニューにある ユーザー を開く。
          • ユーザーを追加 を選択する。

          • ユーザー名 に任意の名前を設定する。
          • 認証情報タイプは アクセスキー を選択する。
          • 次のステップ: アクセス制限 を押す。

          • 既存のポリシーを直接アタッチ を選択する。
          • 検索バーで DynamoDB を入力して探す。
          • AmazonDynamoDBFullAccess を選択する。
          • 次のステップ:タグ を押す。

          • タグは何も入力せず、次のステップ:確認 を押して次の画面に進む。
          • 確認画面に進むので、問題なければ ユーザーの作成 を行う。

          この後の画面で表示される アクセスキーID と シークレットアクセスキー でAWSの認証を行い、DynamoDBが操作できる。

          キーの管理

          キー情報が他の人に漏れるとDynamoDBが好き勝手操作されてしまう。厳重管理すること。
          例えば、プログラムコードにキー情報を書いたままgithubで公開して、とんでも無いことになった事例も散見される。

          # データの登録

          Pythonを使ってデータ登録を行う。 PythonからAWSを操作するためには boto3 を使う。


          • boto3をインストールしていない場合、Tetminalで以下のコマンドを打つ。
          pip3 install boto3

          • Termialからpython3を起動してデータ登録を行う
          import boto3
import datetime

# データベースに接続
session = boto3.session.Session(
    region_name='①[DBのリージョン]',
    aws_access_key_id='②[アクセスキーID]',
    aws_secret_access_key='③[シークレットアクセスキー]'
)
dynamodb = session.resource('dynamodb')

# 登録データの作成
URL = 'https://koz-sec.github.io/dxsec/'
today = int(datetime.date.today().strftime('%Y%m%d'))

item = {
    'URL': URL,
    'date': today,
    'tweeted': False
}

# テーブルに接続
table_name = '④[テーブル名]'
table = dynamodb.Table(table_name)

# データを登録
response = table.put_item(
    TableName=table_name,
    Item=item
)

# 処理結果を確認
if response['ResponseMetadata']['HTTPStatusCode'] != 200:
    print('登録失敗')
    print(response)
else:
    print('登録成功')

          • Terminalで登録成功と表示されたら、DynamoDBの画面に戻る。
          • 画面左メニューの 項目を探索 を選択する。
          • 登録したデータがあれば、無事完了している。

          # データの取り出し

          登録の時に使った put_item() ではなく、get_item() を使えば良い。

          import boto3
import datetime

# データベースに接続
session = boto3.session.Session(
    region_name='①[DBのリージョン]',
    aws_access_key_id='②[アクセスキーID]',
    aws_secret_access_key='③[シークレットアクセスキー]'
)
dynamodb = session.resource('dynamodb')

# 探すデータ
URL = 'https://koz-sec.github.io/dxsec/'

# テーブルに接続
table_name = '④[テーブル名]'
table = dynamodb.Table(table_name)

# データを検索
response = table.get_item(
    Key={
        'URL': URL,
    }
)

# 結果を表示
print(response)

          • 取り出しに成功すると response 変数にデータが保存される。

          # データの更新

          update_item() を使えば良いが、この関数は データが存在しないときに新規作成する という動きをする。そのため、データが存在するかを確認してから更新し、思わぬデータが作成されてしまう事態を避ける。

          また、更新は プレースホルダー を用意して、値を登録するような書き方をする。

          • Key で更新対象を一意に特定する。
          • UpdateExpression でプレースホルダーを設定する。
          • ExpressionAtrrivuteValues で更新後の値を設定する。
          import boto3
import datetime

# データベースに接続
session = boto3.session.Session(
    region_name='①[DBのリージョン]',
    aws_access_key_id='②[アクセスキーID]',
    aws_secret_access_key='③[シークレットアクセスキー]'
)
dynamodb = session.resource('dynamodb')

# 更新するデータ(検索用)
URL = 'https://koz-sec.github.io/dxsec/'

# テーブルに接続
table_name = '④[テーブル名]'
table = dynamodb.Table(table_name)

# データを検索
response = table.get_item(
    Key={
        'URL': URL,
    }
)

# データが存在すればアップデート
if 'Item' not in response:
    print('ない')
else:
    print('ある')
    # 更新(tweetedをFalseに変更)
    rensponse = table.update_item(
      Key= {
        'URL': response['Item']['URL']
      },
      UpdateExpression= 'set tweeted = :FLAG',
      ExpressionAttributeValues= {
        ':FLAG': True
      }
    )

          DynamoDBを確認し、期待通りに更新されていることを確認する。 私の場合、Tweetedの項目が True に変わっていた。
