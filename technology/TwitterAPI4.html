                # Twitter API(4)

                Twitter APIについては、長くなるので章を分けて記載する。

                本章で記載していること

                • AWS(Lambda, DynamoDB)への移行

                # 移行の理由

                プログラムコードをcrontabに登録して定期実行することは可能だと思う。 しかし、それでは作業PCを起動している時間しか動かないので、丸1日動かない日も出てくると思う。

                そこで24時間稼働しているクラウドサービスへの移行を考えた。


                # Lambdaでのプログラム実行

                ここでは、APIキーを環境変数に登録して使用する。 本当は暗号化したキーを使った方が良い。

                • Lambdaを開き、関数を作成する。
                  (参考) AWS(2) Lambda (opens new window)
                • 設定 から 環境設定 を開き、編集 を押す。

                • 環境変数の追加 を押し、APIキーの情報を入力する。キー名は任意である。
                • 暗号化設定は、Lambdaに任せる。

                加えてDynamoDBにURL情報を蓄える機能を実装した。 これは、一度発信したURLを何度も繰り返し投稿しないようにするため、投稿したURLを手元に残して比較するための機能である。

                (参考) AWS(3) DynamoDB (opens new window)


                • コード に移動し、以下のプログラムコードを書く。
                長いので折りたたんでいます(クリックで展開します)

def securityNews():
    
    # 検索キーワード
    all_keywords = [
        ["\"情報漏洩\"OR\"情報漏えい\" exclude:retweets",'情報漏えい']
    ]

    # 認証情報を入力して、
    api_key = os.environ['API_KEY']
    api_secret = os.environ['API_SECRET']
    access_token = os.environ['TOKEN_KEY']
    access_token_secret = os.environ['TOKEN_SECRET']
    
    # Twitterで認証して
    auth = tweepy.OAuthHandler(api_key, api_secret)
    auth.set_access_token(access_token, access_token_secret)
    api = tweepy.API(auth, wait_on_rate_limit = True)
    
    for search_keyword in all_keywords:
        # 検索して
        q = search_keyword[0]
        tweets = api.search_tweets(q=q, tweet_mode='extended', result_type="recent", include_entities=True, count=10)
        
        # DynamoDBに保存＆tweetする
        for a_tweet in tweets:
            if 0 != len(a_tweet._json['entities']['urls']): # URLが存在すれば
                URL = str(a_tweet._json['entities']['urls'][0]['expanded_url'])
                URL = requests.get(URL).url #短縮URLを展開
                if not URL.startswith('https://twitter.com'): #ツイート引用を除外
                    goTweet(URL, api, search_keyword[1])
    
    return

def goTweet(URL, api, search_keyword):
    
    # データベースに接続
    session = boto3.session.Session(
        region_name='ap-northeast-1',
        aws_access_key_id=os.environ['Dynamo_KEY'],
        aws_secret_access_key=os.environ['Dynamo_SECRET']
    )
    dynamodb = session.resource('dynamodb')
    
    # テーブルに接続
    table_name = 'SecurityURLs'
    table = dynamodb.Table(table_name)
    
    # 既にデータがあるか確認(検索)
    response = table.get_item(Key={'URL': URL,})
    
    # データが存在しなければ追加＆ツイート
    if 'Item' not in response:
        # 登録データの作成
        today = int(datetime.date.today().strftime('%Y%m%d'))
        
        item = {
            'URL': URL,
            'date': today,
        }
        # データを登録
        response = table.put_item(
            TableName=table_name,
            Item=item
        )
        # ツイートする
        strTweet = search_keyword + "に関する記事: \r\n" + URL
        api.update_status(strTweet)
    return

                # LambdaにTweepyのインストール

                TweepyのプログラムコードをLambdaにアップロードすることで使用できる。

                ぜい弱性の問題

                プログラムコードにぜい弱性が発見された時、バージョンアップやパッチ適用を行うことで、そのぜい弱性を回避できる。 今回記載する方法(tweepy自体をアップロード)を行うと、その時点のバージョンでプログラムが動き続けてしまう。 今後、ぜい弱性対応などで新しいバージョンが公開された時は、これから行う操作と同様の作業を行い、バージョンを最新に保つ必要がある。

                • 作業PCで、インストール先を指定して Tweepyをインストールする。(-t オプションをつける)
                pip3 install tweepy -t /Users/koz/tweepy

                • tweepyフォルダが作成され、tweepyに関するプログラムが保存された。

                この後の操作

                この後の操作を行うと、lambda関数で既に作ってあるプログラムファイルが全て消え、tweepyのプログラム群で上書きされる。 既に作ったプログラムコードは、ローカルのファイルにコピーして保存する　等の退避を行なってから、次の手順に進む。


                • Tweepyの保存されたフォルダを全て圧縮する。
                • コード画面の右側で、アップロード元 から zipファイル を選択する。

                • アップロード を選択し、先ほどの圧縮フォルダを選択し保存 を押す。
                • tweepyがアップロードされた。

                • 再度、プログラムコードを作って、deploy 後に TEST　から実行する。
                • Twiterアカウントを見ると、期待通り発信されたことを確認できた。

                • Dynamo DBでURL情報が保存されていることを確認できた。(同じURLを何回も発信しないため)

                # 定期実行

                • lambda関数の画面で トリガーを追加 を選択する。
                • EventBridge を選択する。
                • 新規ルールの作成 で1時間ごとに実行されるようルールを作成する。

                cronの時差

                EventBridgeで使われるタイムゾーンはUTCなので、日本のJSTとは9時間の時差がある。
                例えば、朝8時から夕方7時まで１時間ごとに動かしたい場合は、以下のように書く。
                cron(0 23-10 * * ? *)
