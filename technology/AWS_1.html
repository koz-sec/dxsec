          # AWS(1) 個人契約

          # AWSとは

          AWSはAmazonが提供するクラウドサービスである。 サーバ、データベース、ネットワークといったコンピュータリソースがインターネット経由で提供される。

          このようなシステムを自前でサーバを用意して稼働させる場合、設備を買うコストも掛かり、場所も取り、初期設定も色々と必要になる。時々パッチ適用などのメンテナンスも発生する。

          一方、クラウドサービスであれば、そのような煩雑な作業を全部AWSがやってくれるし、場所も取らない。中でも特に良いところは、サブスクリプション(変動費)で契約できる点だと考える。色々試して、上手くいかないことがあれば解約すれば良い。やめる・切り替えるという考えは、自前設備では難しい判断になると思う。

          クラウドは、勉強やチャレンジに向いていると思う。

          • AWSとは (opens new window) / Amazon

          # 契約する

          • AWSのサイト (opens new window) にアクセスする。
          • 今すぐ無料サインアップ を選択する。
          • メールアドレス、アカウント名 を入力して、認証コードをEメールアドレスに送信 を押す。
          • メールで認証コード(数字6桁、10分間のみ有効)が届く。コピーしてブラウザの確認コードに貼り付けし、認証を完了して次へ を押す。
          • アカウントに設定する パスワード を入力し、次へ(ステップ1/5) を押す。


          ■Step 1/5: 利用目的とユーザ情報

          • AWSの利用用途を選ぶ。
            • [　]: ビジネス -職場、学校、組織向け
            • [✔️]: 個人 -ご自身のプロジェクト向け
          • 名前、電話番号、住所 を半角英数字で入力し、Agreementの 合意にチェック をつけて次へ(ステップ2/5) を押す。


          ■Step 2/5: 請求情報

          • Credit Card情報(クレジット番号、有効期限、氏名)を入力し、確認して次へ(ステップ3/5) を押す。

          補足

          ※無料利用の枠を超えない場合は請求しないと記載がある



          ■Step 3/5: 本人確認1

          • 本人確認のため、認証コードを受け取る方法を選ぶ。
            • [✔️]: テキストメッセージ
            • [　]: 音声通話
          • 電話番号 を入力し、セキュリティチェック(画像と同じ文字を入力するゲーム)を行って、SMSを送信する(ステップ4/5) を押す。

          Step 4/5: 本人確認2

          • 入力した電話番号宛に認証コード(数字4桁)が届くので、そのコードをブラウザで入力し、次へ(ステップ5/5) を押す。

          認証コード

          筆者が試した時はすぐ届いたが、認証コードが届くまで10分以上掛かるケースもあるとのこと。



          ■Step 5/5: プラン選択

          • ベーシックサポート(無料) を選んで サインアップを完了 を押す。

          数分待つとアカウントが有効になる旨が表示される。 同じ画面のAWSマネジメントコンソールに進む を押すと、サインイン画面に遷移する。


          # 初めてのサインイン

          • サインイン画面で、ルートユーザ を選び、メールアドレス を入力して 次へ を押す。

          • セキュリティチェック(画像と同じ文字を入力するゲーム)を行って、送信 を押す。
          • アカウント登録時に設定した パスワード を入力して サインイン を押す。
          • サインインに成功すると下記の画面が表示される。

          # MFA設定

          登録後、至急行うべきセキュリティ強化作業として MFA(多要素認証: Multi Factor Authentication) を実施する。

          多要素認証が必要な理由

          ユーザIDとパスワードだけでは、サイバー攻撃者に何度もサインイン操作を試された結果、サインインされることもありうる。 仮にアカウントが取られてしまうと、大量のサーバを立ち上げて仮想通貨のマイニング等に使われることもある。 これで発生した莫大な使用料金は 契約者(あなた)宛に請求 される。 1日数万円、1週間あれば数百万円にも達するだろう。

          そこで、多要素認証を使う。あなたのスマートフォンを持っている人しかサインインできないように設定する。



          • 画面上の サービス を押し、セキュリティ、ID、およびコンプライアンス から IAM を押す。

          • rootユーザのMFA追加が推奨されているので、MFAの追加 を押す。
          • セキュリティ認証情報のページが開く。MFA有効化 を押す。
          • 下図のようにMFAの名前(好きな名前)を入力し、仮想MFAデバイス を選んで 続行 を押す。

          • MFAデバイスの設定画面が表示される。
          • QRコードの表示を押すと、QRコードが表示される。
          • 普段使っているスマートフォンに Microsoft authenticator(以下、Authenticator) をインストールする。

          使用できる多要素認証アプリ

          私はMicrosoftを使用しているが、他にもGoogle等の認証アプリもある。 下記リンク先で記載のあるものから、好きなものを使用すれば良い。 互換性のあるアプリケーションリスト (opens new window) / AWS

          • スマートフォンでAuthenticatorを起動し、検証済みID を選択する。
          • QRコードをスキャン を押すとスマートフォンが写真撮影画面になる。
          • ブラウザに表示されたQRコードをスキャンする。
          • AuthenticatorにAmazon Web Serviceが登録される。

          • ブラウザに戻り、Authenticatorに表示される6桁の数字を入力する。
          • MFAの割り当て を選択する

          設定がうまくできたか、動作確認をする。

          • AWSからサインアウトする。
          • コンソールにサインインを押す。
          • メールアドレス、セキュリティチェック(画像と同じ文字を入力するゲーム)、パスワードを入力し、サインインを押す。
          • 多要素認証の画面が表示されたら設定されている。Authenticatorの6桁数字を入力してサインインできたら、うまく設定できている。

          # 料金の監視設定

          AWSは使用した分だけ請求が発生してしまう。 処理量が増えて想定よりコストが増えた場合に備え、請求額に応じてアラートが発出されるように設定する。

          • アカウント名 を展開し、請求ダッシュボード を開く。

          • 画面左のメニューで 請求設定 を開く。

          • チェックをつける

            • [✔︎]E メールで PDF 版請求書を受け取る
            • [✔︎]無料利用枠の使用アラートを受信する
              • メールアドレスを入力する。
            • [✔︎]請求アラートを受け取る

          • 詳細設定を保存 を選択する。

          • 画面左のメニューから Badgets を開く。

          • 予算の作成 を開く。

          • 以下のように設定する。

            • 予算設定: テンプレートを使用(シンプル)
            • テンプレート: ゼロ支出予算
            • 予算名: 好きな名前をつける。
            • Eメール受信者: 通知先のアドレスを記入する。

          • 予算を作成 を押す。


          無料使用枠を超えた時(厳密には、$1を超えた時)に通知されるように設定された。
