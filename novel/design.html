            # 問題の発生 工場

            三ツ星工業

            10月。だんだん寒くなってきた。
            ３０周年
            工場に品質検査機器
            正常品と異常品を自動判定

            # ある時、判定が逆に。
            再セットアップしても、繰り返し問題が起きる
            サイバー攻撃を疑う
            本社に連携

            # 調査 工場

            FWのパケットを確認するも異常なし
            ベンダーの製品に不具合があるのでは

            現場のスイッチで確認
            見知らぬ機器からの通信
            管理者パスワードが漏洩している恐れ
            一斉棚卸し
            IoT機器ストロベリーパイを発見

            昨日、設備を設置した時に証拠写真を撮っていた。
            その時、すでに設置されていたことが発覚。

            日々のメンテナンス作業はあるものの、物理的に写真を撮るのは稀。
            ５営業日前の作業でネットワーク機器の設置をしたときに写真を撮っていた。
            その時、同じ場所にストロベリーパイはなかった。

            つまり、この1週間、むしろ検査システムの設置直前に設置されたと考えるのが妥当

            事業所内の誰も知らない
            そもそも、そんな杜撰な資産管理をしていたとは。

            工場長から取り外す許可を得る
            不具合が改善
            さっそく本社とリモート会議を繋ぎ、ストロベリーパイの分析に入る。

            # しかし、そもそも一体誰がストロベリーパイを設置したのだろうか。

            # 過去の悪事 本社

            8月。本社では。

            ３０周年に向けた準備として、猪部と岩本が検査システム導入を準備

            創立パーティを9月の頭に控え、佳境に差し掛かっている

            終電がもうすぐ。
            運用のドキュメントだけ、今週中に仕上げたい。
            続きは帰ってやるか。

            USB挿そうとしてエラー
            制限が掛かっているから

            クラウドに上げることを決めた
            「なんだよ、そのクラウドサービス。」
            猪部と、岩本しか知らないURLで共有

            運用ドキュメントと、いくつかのファイルを封入
            念のため、パスワードを付けた圧縮フォルダで。
            「これなら大丈夫だろ？」

            # 調査結果 本社

            10月。
            ストロベリーパイの調査結果が出る。

            パスワードだけではない。
            三ツ星の環境に合わせて特注で作り込んだ部分をピンポイントで狙っている。
            確かに、稼働してからシステムの通信範囲でパケットキャプチャを使えば、攻撃は可能だと思う。
            だが、動き始めてまだ2日目。入退管理もしっかりしている

            # この攻撃の狙い方は、まさに運用ドキュメントを読んだ人間しか考えられない。
            疑いの目は、設計や構築を行なった猪部と岩本に向けられる。
            あるいはオペレーションをしている工場勤務社員16名。

            # 情報漏洩 本社

            8月。
            情報漏洩発覚の電話
            社内機密のデータを印刷したものが封筒で届いた
            法務より、EU圏の情報が漏洩していたらGDPR違反
            違反者には「年間売上の⚪︎%」もの制裁金
            タイムリミットは72時間。それまでに当局に報告しないと。

            猪部は他の誰にもURLを教えていない
            岩本か？

            # 岩本も知らない。

            岩本は個人メールに転送していた
            しかし、メールはスマートフォンからしかアクセスしない
            スマートフォンにもロックが掛かっている

            メールに不正アクセスを疑う
            アクセスあると通知
            でも、そのようなメールは..
            9月12日に都内からアクセス
            心当たりなし
            IPアドレスはクラウドプロキシ。
            三ツ星かもしれないし、他の会社かもしれない。
            法的にIPアドレス開示請求をすると数ヶ月から数年かかることも。
            間に合わない

            でもログアウトしたはず。

            共用PCを開きブラウザを立ち上げる
            Doobleメールのログイン画面では、前回ログインのIDが残っている
