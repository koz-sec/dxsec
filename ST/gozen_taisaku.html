# ITストラテジスト午前1,2の対策

知識を問われるので一定水準の勉強をしていることが必要である。
問題は難しくはないが、合格ラインの6割を取れなければ午後の得点が良くても採点されない。 目安として、過去問で8割を取れるくらいの対策がちょうど良いだろう。


# 午前1対策

午前1は過去問の流用がほとんどを占める。 新しい問題を作らないのはIPAの怠慢ではないだろうか。
IPAに確認したところ、基本的な知識の有無として、ちゃんと勉強しているかを確認するのが午前1の意図なのだとか。まじか。


さらに、高度試験の午前1は共通(全部同じ)である。応用情報技術者試験の午前問題が、試験問題そのもの 試験範囲である。


こういう背景もあり、以下のサイトを繰り返し解いていれば合格点の6割を突破できる。
「全く同じ問題」が出る訳なので、10年分くらいを解けば「ああ、解いたことある」が山ほど出てくる。本番で8割9割の得点も夢ではない。

  • 過去問道場 (opens new window) / 応用情報技術者試験ドットコム

# 午前1の免除制度

なお、午前1は条件を満たすことで免除され、午前2から受講できる制度もある。

  • 試験の一部（午前Ⅰ試験）免除制度 (opens new window) / IPA

# 午前2対策

午前2も午前1と同様に過去問の流用が多い。

過去問を整理してくれているサイトがあるので活用すると良い。
ITストラテジスト過去問対策.com (opens new window)


# 対策時期

試験に受かるだけが目的であれば、直前に解いて短期記憶で正答を導く方法が効率的である。 試験前日に解いた問題がそのまま出るのだから、間違える訳が無い。
