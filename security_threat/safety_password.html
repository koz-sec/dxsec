                # 安全なパスワードの設定

                多くのシステム、特にウェブサービスやクラウドサービスにおいては、利用する際にログインが求められる。 ログインすることによって、自分専用のページや、自身が登録した情報にアクセスすることができる。 当然ながら、IDとパスワードが他の人にバレれば不正アクセスを受け、自身の情報が悪用される恐れがある。


                # パスワードを破ろうとするサイバー攻撃

                攻撃者はパスワードを手入力するのではなく、ツールを使って大量のリクエストを送信する。 よく知られる攻撃には、ブルートフォース(総当たり)攻撃、パスワードリスト攻撃、パスワードスプレー攻撃 等がある。

                近年は攻撃者が使うツールの性能が向上しており、平易なパスワードは数秒で破られてしまう。例え複雑なパスワード(数字と英字と記号を混ぜたもの)であっても、8桁程度ならわずか2日で解析されてしまうという報告もある。


                # （対策１）初期パスワードの変更

                機器をインターネットに接続する場合、その機器がインターネットを介して狙われることを想定すべきである。
                近年は、パソコンやスマートフォンだけではなく、IOT機器、ハンディターミナル、防犯カメラ等もインターネットにつながっている。 こういった機器の初期パスワードは製品マニュアルに書かれていることもあり、購入後に変更しなければ、誰でも不正アクセスができてしまう。

                DANGER

                パスワードを初期状態のままで使うことは危険なので、必ず変更すること。


                # （対策２）長いパスワードの設定

                パスワードは桁数が増える方が強度が増し、12桁を超えるようなら解析に100年を超えるというデータもある。

                しかし、長いパスワードだと覚えられないという人は多い。その場合、IPAの チョコっとプラスパスワード (opens new window) 等を参考にすると良い。

                WARNING

                自分のパスワードの強度を調べる HOW IS MY PASSWORD? (opens new window) というサイトがある。しかし、 ここに入力したパスワードがこのサイトから漏洩する 可能性はある。セキュリティ専門家として、ここに実際のパスワードを入力すべきでないことを警告する。


                # （対策３）パスワード使い回し

                多くのクラウドサービスでIDにメールアドレスが使われている。複数のサービスでパスワードを使い回ししていると、万が一に使っているサービスでパスワードが漏れたら、他のウェブサービスに全く同じID・パスワードでログインされてしまう。
                従って、なるべくサービスごとに違うパスワードを使うことが望ましい。自分で決めたパスフレーズの後ろに使用するサービス名を付け加えるだけでも、ツールを使った不正ログインからアカウントを守ることができる。
                Gmailの例：kozP@ssword#Gmail
                Amazonの例：kozP@ssword#Amazon


                # （その他）パスワードの定期変更

                パスワードが漏洩したら悪用されてしまう。では、時々変更すれば侵害を受ける可能性が低くなるのだろうか？


                総務省の案内 (opens new window) によると、パスワードの定期変更は しなくて良い と書かれている。

                パスワードを複数のサービスで使い回さない（定期的な変更は不要）

                このような案内の背景には、パスワードを定期的に変更すると、変更のせいでパスワードを忘れてしまったり、忘れないために簡単なパスワードを使ってしまう、といった別の問題が発生することが挙げられる。 定期的な変更ではなく、強固なパスワードを使用するした方が良いと考えられている。


                これは日本独自の考え方という訳ではなく、米国NISTの SP800-63B (opens new window) という認証に関するガイドラインにも同様の記載がある。

                Verifiers SHOULD NOT require memorized secrets to be changed arbitrarily (e.g., periodically)​
                (意訳)認証する人が覚えているパスワードを(定期的に)変更するよう要求すべきではない。

                実は日本は2018年までパスワードの定期変更を推奨していた。米国NISTのガイドライン改定が2017年に行われ、日本も追従する形で総務省のガイドラインが変更されている。


                しかし、あらゆる場面でパスワード変更不要な訳ではない。例えば、FDA(米国食品医薬品局)が定める CRF21 (opens new window) のような例外もある。

                Ensuring that identification code and password issuances are periodically checked, recalled, or revised (e.g., to cover such events as password aging).
                (意訳)IDとパスワードが定期的に確認または改定されていることが必要です。

                食品・医薬品の業界ではパスワードの定期変更がいまだに求められており、これを無視して定期変更をしなければ、FDAが定める法規に違反してしまう。


                パスワード定期変更の要否は事業環境によっても左右されるため、ご自身の環境を踏まえた上で、適切な方法でアカウントを守りたい。


                # パスワードリマインダー

                ウェブサービスで見られる「パスワードを忘れた方はこちら」と書かれた機能は注意したい。簡単な質問であれば、SNS等で公開されている情報を集めることで、見知らぬ人でも「あなただけの答え」を入力できてしまう。以下のような方法でパスワードを抜き取られることが考えられる。

                攻撃イメージ

                1. メールアドレスや電話番号を使いFacebookアカウントを見つける
                2. 出身地やペットの名前といった情報を集める
                3. ショッピングサイトで「パスワードを忘れた方はこちら」から質問に答える
                4. パスワードがリセットされ、攻撃者の手に渡る

                今一度、「秘密の質問」に対して真面目な回答をすべきなのか、考えてほしい。
